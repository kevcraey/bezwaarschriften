# Design: Extractie-detail side-panel

## Doel

Na extractie van bezwaren uit een document, kan de gebruiker via een vergrootglas-knop in de bezwaren-tabel een side-panel openen dat de details toont: per bezwaar de LLM-samenvatting en de originele passage uit het bezwaarschrift.

## Database schema

Twee nieuwe tabellen met cascade delete naar `extractie_taak`:

```sql
CREATE TABLE extractie_passage (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    taak_id     BIGINT NOT NULL REFERENCES extractie_taak(id) ON DELETE CASCADE,
    passage_nr  INT NOT NULL,
    tekst       TEXT NOT NULL
);

CREATE TABLE geextraheerd_bezwaar (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    taak_id     BIGINT NOT NULL REFERENCES extractie_taak(id) ON DELETE CASCADE,
    passage_nr  INT NOT NULL,
    samenvatting TEXT NOT NULL,
    categorie   VARCHAR(50) NOT NULL
);
```

## Backend

### Entiteiten (package `project`)

- `ExtractiePassageEntiteit` -- JPA entity voor `extractie_passage`
- `GeextraheerdBezwaarEntiteit` -- JPA entity voor `geextraheerd_bezwaar`

### Repositories

- `ExtractiePassageRepository` -- `findByTaakId(Long taakId)`
- `GeextraheerdBezwaarRepository` -- `findByTaakId(Long taakId)`

### Opslag bij extractie

- `ExtractieTaakService.markeerKlaar()` uitbreiden: accepteert nu het volledige `ExtractieResultaat` en slaat passages + bezwaren op naast de counts.
- `ExtractieWorker.verwerkTaak()` geeft het volledige `ExtractieResultaat` door in plaats van alleen counts.

### Nieuw API endpoint

`GET /api/v1/projects/{naam}/extracties/{bestandsnaam}/details`

Zoekt de meest recente extractie-taak met status KLAAR voor die bestandsnaam en retourneert:

```json
{
  "bestandsnaam": "bezwaar-001.txt",
  "aantalBezwaren": 5,
  "bezwaren": [
    {
      "samenvatting": "Vrees voor geluidsoverlast door evenementen",
      "passage": "De letterlijke tekst uit het bezwaarschrift..."
    }
  ]
}
```

Elk bezwaar bevat de samenvatting + de passage-tekst (gejoind via passage_nr).

## Frontend

### Vergrootglas-knop (bezwaarschriften-bezwaren-tabel.js)

- `vl-button-icon` met `icon="search"` en `ghost` attribute
- Zelfde stijl als de bestaande delete-knop, maar zonder `error`
- Alleen gerenderd wanneer `rij.status === 'extractie-klaar'`
- Naast de bestaande delete-knop in de actie-kolom
- Dispatcht custom event `toon-extractie-detail` met `{ bestandsnaam }` in detail

### Side-panel (bezwaarschriften-bezwaren-tabel.js)

- Hergebruikt het `vl-side-sheet` patroon uit bezwaarschriften-kernbezwaren.js
- `hide-toggle-button` attribute, header met titel + sluit-knop (x), scrollbare body
- `margin-right: 33.3%` shift op de host wanneer open (`:host(.side-sheet-open)`)
- Header: documentnaam + "X bezwaren gevonden"
- Body per bezwaar:
  - **Samenvatting** (vetgedrukt)
  - **Passage** (italic, ingesprongen -- zelfde stijl als passages in kernbezwaren)

### Data ophalen

Bij klik op vergrootglas:
`fetch(/api/v1/projects/${project}/extracties/${bestandsnaam}/details)`

Response vult het side-panel.

## Beslissingen

- Categorie wordt opgeslagen maar niet getoond in het side-panel (kan later)
- DocumentSamenvatting wordt niet getoond
- Vergrootglas-knop alleen zichtbaar bij status `extractie-klaar`
