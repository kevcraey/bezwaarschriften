# TDR-003: pgvector voor vector similarity search

**Datum:** 2026-01

## Context

Bezwaren worden als embeddings opgeslagen en vergeleken via cosine similarity. Een dedicated vector-database (Pinecone, Weaviate) zou extra infrastructuur vereisen.

## Besluit

PostgreSQL + pgvector extensie. Embeddings opslaan in `vector` kolommen, cosine similarity queries via `<=>` operator. Eén database voor zowel relationele data als vectoren.

## Consequentie

- Geen extra database-infrastructuur.
- PostgreSQL is al aanwezig voor de rest van de applicatie.
- Performantie voldoende voor verwacht volume (duizenden, niet miljoenen vectoren).
