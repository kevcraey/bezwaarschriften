# ADR-002: WebSocket voor real-time taakupdates

**Datum:** 2026-02

## Context

Extractie- en consolidatietaken duren seconden tot minuten. De gebruiker moet de voortgang live zien zonder de pagina te verversen.

## Besluit

Spring WebSocket (`TaakWebSocketHandler`) stuurt JSON-berichten bij elke statuswijziging van een taak. Drie berichttypen: `taak-update` (bezwaar-extractie), `consolidatie-update`, `tekst-extractie-update`. Frontend luistert via native `WebSocket` API.

## Consequentie

- Directe feedback in de UI zonder polling.
- Server houdt actieve sessies bij (in-memory `ConcurrentHashMap`).
- Bij server-restart verliezen clients de verbinding; frontend reconnect automatisch.
