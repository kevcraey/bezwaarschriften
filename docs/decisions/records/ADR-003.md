# ADR-003: Persistente bestandsstatus in database

**Datum:** 2026-03

## Context

Bestandsstatus werd bijgehouden in een in-memory `ConcurrentHashMap` per sessie. Bij server-restart of meerdere instanties ging status verloren.

## Besluit

Bestandsstatus opslaan in de `bezwaar_bestand` tabel via JPA-entiteit `BezwaarBestandEntiteit`. Status-enum `BezwaarBestandStatus` met prefixen (`TEKST_EXTRACTIE_*`, `BEZWAAR_EXTRACTIE_*`) weerspiegelt de volledige verwerkingspipeline.

## Consequentie

- Status overleeft server-restarts.
- Geschikt voor multi-instance deployments.
- Liquibase-migratie nodig voor nieuw schema.
- Vervangt de eerdere in-memory oplossing volledig.
