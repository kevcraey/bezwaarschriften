# TDR-002: Spring AI met profile-based provider switching

**Datum:** 2026-01

## Context

Ontwikkeling gebeurt lokaal met Ollama; productie draait op OpenAI. Beide providers moeten naast elkaar bestaan zonder code-aanpassingen.

## Besluit

`@Profile("openai")` en `@Profile("local")` scheiden de AI-configuratie. Profile-specifieke `application-*.properties` excluderen de andere auto-config. `RetryConfiguration` is profiel-onafhankelijk.

## Consequentie

- Wisselen tussen providers via één environment variabele.
- Geen conditionele logica in domeincode.
- Twee sets configuratie-bestanden te onderhouden.
