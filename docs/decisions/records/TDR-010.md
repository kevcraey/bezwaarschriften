# TDR-010: Minimaal mocken in tests

**Datum:** 2026-01

## Context

Uitgebreid mocken verbergt integratiefouten. Bugs slippen door wanneer mocks niet het echte gedrag weerspiegelen.

## Besluit

Voorkeur voor integratietests met echte dependencies (Testcontainers). Mocks enkel voor externe services (AI-providers) en waar strikte isolatie noodzakelijk is (controller-tests).

## Consequentie

- Meer bugs gevonden tijdens testen.
- Tests zijn trager maar betrouwbaarder.
- Testcontainers en Docker zijn vereist voor de meeste tests.
