# ADR-004: Async workers voor langlopende taken

**Datum:** 2026-02

## Context

AI-extractie en consolidatie zijn langlopende operaties (seconden tot minuten per bestand). Synchrone verwerking zou de HTTP-thread blokkeren.

## Besluit

`@Async` workers (`ExtractieWorker`, `TekstExtractieWorker`) verwerken taken op achtergrondthreads. Controller dient taken in en retourneert direct. Workers melden voortgang via WebSocket.

## Consequentie

- HTTP-requests keren direct terug.
- Retry-logica in de worker bij tijdelijke AI-fouten.
- Complexere foutafhandeling (fouten worden asynchroon gemeld via WebSocket).
